## 🧭 About Me

I design and build **robust, maintainable backend systems** with ASP.NET Core, C#, and SQL Server — with a deliberate focus on architectural integrity and long-term code health.

Architecture, for me, is not a template. **Onion / Clean Architecture, CQRS, and Domain-Driven Design** are tools for managing complexity as requirements evolve — from domain modeling to DI configuration, every layer gets the same analytical discipline.

- 🔨 Currently building **Horis Geneve** — a multi-tier partnership & investment platform (freelance engagement)
- 🧅 Author of **OnionSetUp** — a production-ready Onion Architecture boilerplate for .NET 9
- 🎯 Core focus: RESTful API design · EF Core · JWT & Identity · layered, extensible solutions
- 📚 Currently deepening: CQRS at scale, testing strategies, containerization
- 🌍 Azerbaijani (native) · Turkish (fluent) · English (intermediate)
- 📫 **mvusal316@gmail.com**

## ⚙️ How I Build Software

| Principle | In practice |
|-----------|-------------|
| **Decompose before implementing** | I identify structural risks early and break problems down before writing code |
| **Patterns as tools, not dogma** | CQRS, Repository, Options Pattern — applied where they manage complexity, never for their own sake |
| **Consistency reduces rework** | Uniform structure across the codebase keeps it readable under real engineering pressure |
